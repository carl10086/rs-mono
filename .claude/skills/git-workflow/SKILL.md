# Git 工作流自动化

## 核心要点

1. 使用中文
2. 简短
3. 自动化执行，减少人工干预

## 工作流程

### 1. 分析变更
执行 `git status` 查看当前变更，分析变更类型：
- **feature/**: 新功能
- **fix/**: Bug 修复
- **refactor/**: 重构
- **docs/**: 文档
- **style/**: 格式
- **test/**: 测试
- **chore/**: 构建/工具

### 2. 智能分支命名
规则：`{类型}/{简短描述}-{日期}`

### 3. 分支管理
- 从当前分支创建新分支
- 分支已存在则切换

### 4. 提交代码
- `git add <具体文件>`
- 生成符合 commitizen 格式的 commit message
- `git commit`

### 5. 自动推送
- `git push -u origin {分支名}`
- 显示分支链接

## 特殊情况

- **分支已存在**：切换到已有分支
- **冲突**：提示用户解决
- **空提交**：跳过
- **推送失败**：重试一次，仍失败则提示
